# ItJima Personal Alpha — Mobile QA & 5-User Test

> 기준 브랜치: `codex/personal-alpha`  
> 테스트 범위: 게스트 모드, 모바일 브라우저  
> 목적: 기능의 수가 아니라 `맡기기 → 오늘 → 보관함`의 안심 루프가 처음 보는 사람에게도 통하는지 확인한다.

## 1. 테스트 전 8분 모바일 QA

5명을 부르기 전에 실제 휴대폰에서 아래 항목을 한 번만 확인한다. 운영 DB 마이그레이션 없이 검증할 수 있도록 로그아웃된 게스트 모드를 사용한다.

- [ ] Preview가 열리고 `맡기기 · 오늘 · 보관함` 세 탭이 보인다.
- [ ] 화면이 가로로 밀리거나 잘리지 않고, 하단 버튼이 안전 영역 안에 있다.
- [ ] 날짜가 없는 생각을 맡기면 `안전하게 맡아뒀어요`와 `추천 시점`이 나타난다.
- [ ] `이때 다시 꺼내기`, `시점 바꾸기`, `보관만 하기`, `일단 맡겨두고 나중에 정할게요`가 각각 동작한다.
- [ ] 현재보다 이른 시점으로 설정한 생각이 `오늘`에 나타난다.
- [ ] `처리했어요`, `내일로 미루기`, `이제 괜찮아요`가 각각 중복 없이 한 번만 처리된다.
- [ ] 처리한 생각과 보관만 한 생각을 `보관함`에서 검색할 수 있다.
- [ ] 새로고침해도 맡긴 생각과 처리 상태가 유지된다.
- [ ] 실제 기기에서 본문 폰트, 아이콘, 이미지가 깨지지 않는다.

하나라도 데이터가 사라지거나 핵심 과업을 진행할 수 없으면 모집 전에 수정한다. 미세한 간격이나 취향 차이는 기록만 하고 테스트를 진행한다.

## 2. 참여자 구성

제품 제작에 참여하지 않은 5명을 모집한다.

- 자신에게 카카오톡 보내기나 메모 앱으로 생각을 남기는 사람 2명
- 리마인더·캘린더·할 일 앱을 쓰는 사람 2명
- 별도 도구 없이 머리로 기억하거나 자주 잊는 사람 1명

가능하면 iPhone과 Android를 모두 포함한다. 한 사람당 15~20분, 화면 공유 또는 대면 관찰로 진행한다.

## 3. 섭외 문구

```text
제가 만든 모바일 앱 ItJima를 처음 보는 상태로 15분 정도 써보고
솔직하게 말해주실 분을 찾고 있어요.

생각나는 것을 빠르게 맡겨두었다가 필요한 순간 다시 보는 앱이에요.
사용자를 평가하는 테스트가 아니라 앱에서 헷갈리는 부분을 찾는 테스트라
잘하려고 하실 필요는 전혀 없어요.

아이폰이나 안드로이드 브라우저만 있으면 됩니다.
가능한 시간 알려주시면 링크를 보내드릴게요.
```

## 4. 진행 원칙

시작할 때 아래 문장만 말한다.

```text
제가 만든 화면이라 어디를 눌러야 하는지 설명하지 않을게요.
보이는 것과 기대하는 것을 편하게 소리 내어 말해주세요.
막혀도 사용자 문제가 아니라 화면 문제입니다.
```

- 버튼 이름을 알려주거나 손가락으로 위치를 가리키지 않는다.
- 참여자가 10초 이상 완전히 멈췄을 때만 “지금 무엇을 찾고 있나요?”라고 묻는다.
- 설명보다 행동, 첫 탭, 망설임, 실제 표현을 기록한다.
- 기능 아이디어가 나오면 바로 약속하지 않고 이유와 발생 맥락을 묻는다.
- 화면 녹화나 음성 녹음을 할 경우 시작 전에 동의를 받는다.

## 5. 테스트 과업

과업은 한 번에 하나씩 읽어준다. 괄호 안 내용은 진행자만 본다.

### 과업 1 — 첫 맡기기

> 내일 아침에 꼭 다시 떠올리고 싶은 실제 생각 하나를 이 앱에 맡겨보세요.

관찰: 입력까지 걸린 시간, 추천 시점의 의미를 이해하는지, 승인과 변경 중 무엇을 고르는지.

### 과업 2 — 날짜 없는 생각

> 언제 다시 볼지는 모르지만 잊고 싶지 않은 아이디어나 참고할 내용을 하나 맡겨보세요.

관찰: 날짜가 없을 때 앱이 무엇을 했다고 이해하는지, 보관과 나중 결정의 차이를 예측하는지.

### 과업 3 — 지금 필요한 생각 처리

> 지금 다시 떠올라야 하는 생각 하나를 만들어 `오늘`에서 확인한 뒤, 처리했다고 가정해 마무리해보세요.

필요할 때만 “추천 시점을 현재보다 이른 시간으로 바꿔도 됩니다”라고 말한다.

관찰: `오늘`을 찾아가는지, 핵심 카드와 버튼을 발견하는지, 처리 후 어디에 남을 것으로 예상하는지.

### 과업 4 — 미루기 또는 놓아주기

> 또 다른 생각을 `오늘`에 나타나게 한 뒤, 이번에는 내일로 미루거나 이제 필요 없다고 처리해보세요.

관찰: 두 선택의 차이가 명확한지, 누른 뒤 결과에 확신이 있는지.

### 과업 5 — 다시 찾기

> 방금 처리했거나 보관한 생각 하나를 다시 찾아보세요.

관찰: 어느 탭을 먼저 고르는지, 검색을 발견하는지, 완료한 생각이 보관함에 남아 있는 것이 자연스러운지.

## 6. 세션 후 질문 3개

1. 이 앱을 친구에게 한 문장으로 설명한다면 뭐라고 할 것 같나요?
2. 어느 순간 가장 안심됐고, 어느 순간 가장 헷갈리거나 불안했나요?
3. 내일부터 실제로 쓴다면 어떤 생각을 가장 먼저 맡길 것 같나요?

## 7. 관찰 기록 양식

| 참여자 | 기기 | 과업 | 결과 | 소요 시간 | 첫 행동 | 막힌 지점·실제 발화 | 예상과 실제의 차이 | 심각도 |
|---|---|---:|---|---:|---|---|---|---|
| P1 | iPhone / Android | 1 | 무도움 성공 / 도움 성공 / 실패 | 00:00 |  |  |  | P0–P3 |

심각도 기준:

- `P0`: 데이터 유실, 크래시, 진행 불가
- `P1`: 핵심 루프를 혼자 완료하지 못함
- `P2`: 완료는 하지만 의미를 잘못 이해하거나 오래 망설임
- `P3`: 문구·간격·취향 수준의 불편

## 8. 개인 알파 통과 기준

다음을 모두 만족하면 기능 추가 없이 개인 알파를 닫고 포트폴리오 정리로 넘어간다.

- 5명 중 4명 이상이 과업 1, 3, 5를 도움 없이 완료한다.
- 5명 중 4명 이상이 ItJima를 캘린더나 할 일 목록이 아니라 “생각을 맡겼다가 다시 보는 앱”으로 설명한다.
- 첫 맡기기 이후 두 번째 맡기기는 4명 이상이 3초 안에 시작한다.
- 데이터 유실과 중복 생성이 0건이다.
- 같은 P1 문제가 2명 이상에게 반복되지 않는다.

한 명의 기능 요청만으로 새 기능을 만들지 않는다. 두 명 이상에게 반복된 행동 문제를 우선하고, 해결도 기존 여정 `맡기기 → 오늘 → 보관함` 안에서 가장 작은 변경으로 한다.
